### 역할이란?



IAM에서 사용해 볼 수 있는 역할은 특정 IAM 사용자에게 사용자가 가지고 있지 않은 권한을 임시로 부여할 때 활용될 수 있습니다.

또한 이번 글에서 다루진 않겠지만 역할은 임시 권한 부여뿐만 아니라 EC2, Lambda와 같은 AWS 서비스에서 다른 AWS 서비스 자원에 접근하려고 할 때 사용될 수도 있습니다.

 

사용자에게 역할을 부여할 때 주의할 점이 있습니다.

특정 IAM 사용자에게 부여된 권한을 이용하여 AWS 서비스를 사용하고 있을 때 해당 사용자에게 역할을 부여해 버리면 기존에 부여되어 있던 권한을 사용할 수 없게 됩니다.

즉, 역할에 적용되어 있던 권한만 사용될 수 있게 되서 기존에 다른 권한이 부여되어 있던 사용자라면 적용할 때 한번 더 확인하여 적용해 볼 필요가 있습니다.



### 역할 사용 방법



[ 1. IAM 접속 ]

<img src="/images/fda54f7dc13d77898347d036d6c4d147.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



[ 2. 역할 만들기 ]

왼쪽에서 "역할" 메뉴를 선택한 뒤 오른쪽에서 "역할 만들기" 버튼을 클릭합니다.

<img src="/images/6bd7a8dde72da10f298d03e7d311bb8e.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



그리고 다음과 같이 원하는 설정 값을 입력해주시면 됩니다.

먼저 "엔터티 유형"은 AWS 계정으로 선택해주셔야 합니다.

그 아래 계정 및 옵션은 상황에 맞게 적절하게 설정해주시면 됩니다.

<img src="/images/4530f335920f180d019b4cb3af6d9021.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



다음은 역할에 부여할 권한을 추가해주시면 됩니다.

저는 테스트를 위해 S3 접근 권한만 부여하도록 하겠습니다.



<img src="/images/1f3dfd990067013cf302740ee7b838d0.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />

 

마지막으로 역할 이름과 설명을 입력한 뒤 최종 생성해 줍니다.

<img src="/images/79544617c04611ecdc392e663119988b.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



[ 3. STS 정책 생성 ]

역할 전환을 하기 위해서는 사용자에 STS Assume 정책이 적용되어 있어야 합니다.

정책 등록을 하기 위해 가장 먼저 생성했던 역할의 ARN 정보를 카피해 줍니다.

<img src="/images/bbf03726bb92b657d2323ad53be8d740.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



그리고 왼쪽에서 "정책" 메뉴를 선택한 뒤 오른쪽에서 "정책 생성"을 클릭합니다.

<img src="/images/06bd9c17903f082260d18c25ddae84c0.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



다음으로 가장 먼저 서비스에 sts를 검색한 뒤 sts를 선택해 줍니다.

<img src="/images/0d995460190a2fd0c3a5ca439a45b00b.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



"쓰기" 에서는 AssumeRole을 선택해 주고 "ARN 추가"를 클릭하여 위에서 생성했던 역할을 다음과 같이 추가해 주시면 됩니다.

<img src="/images/cc09af35988d8e762dff00ac45319138.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



마지막으로 이름과 설명을 입력한 뒤 최종 생성을 합니다.

<img src="/images/b719540212ac839cde56f2b8756bd785.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



[ 4. 사용자에 STS 정책 적용 ]

역할 전환을 원하는 사용자의 세부 정보로 넘어와 오른쪽에 있는 "권한 추가"를 클릭합니다.

<img src="/images/94699d2e0a04b18712129db236b13b10.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



그리고 다음과 같이 위에서 생성한 STS 정책을 추가해 줍니다.

<img src="/images/51fa512b4804019f08a00343315d484a.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



[ 5. 역할 전환 ]

역할이 생성되었다면 역할 전환을 해보겠습니다.

역할 전환을 하기 위해서는 역할 전환을 수행할 IAM 사용자 계정에 접속을 해줘야 합니다.

역할을 부여하고 싶은 IAM 사용자 계정에 접속한 뒤 다음과 같이 우측 상단에서 "역할 전환"을 클릭합니다.

<img src="/images/a2debbfaa905ebf9dba2599eeb40608a.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



다음으로 나오는 화면에서도 "역할 전환"을 클릭합니다.

<img src="/images/91ec259ca0255fb2b67a153881a502c9.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



다음 화면이 보이면 계정 번호와 적용할 역할 이름을 입력해 주시면 됩니다.

<img src="/images/54e8a18a2d46a1c15c3eb4c5cc89e6b9.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



### 테스트



역할 전환이 성공적으로 되었다면 역할이 부여된 계정에 접속했을 때 다음과 같은 표시를 확인해 볼 수 있습니다.

<img src="/images/0aaf38042493d4a642a939210c1269df.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />

 

부여된 역할은 S3만 사용이 가능하기에 다음과 같이 S3에 접속했을 때 아무 문제 없이 사용이 가능한 것을 확인할 수 있습니다.

<img src="/images/5aef5e3c86ec9cc248e3c0b26ce62999.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />



반대로 기존에 사용자가 사용 가능했던 EC2에 접속해 보면 다음과 같이 권한이 적용되어 있지 않을 것을 확인할 수 있습니다.

위에서 말한 것처럼 기존에 권한이 있더라도 역할이 부여되면 부여된 역할이 가지고 있는 권한만 적용되기에 사용하지 못하는 상황입니다.

<img src="/images/568b324ee2dfc2caa8b3339946a41fd7.jpg" alt="file" width="800" height="400" style="max-width: 100%; height: auto;" />
